2.  PRESENTACIÓN
Laravel es uno de los frameworks PHP más populares y elegantes del mundo del desarrollo web. Creado por Taylor Otwell en 2011, su filosofía es hacer que el desarrollo sea una experiencia agradable sin sacrificar la funcionalidad. En esta guía aprenderás desde cero: cómo instalar el entorno, cómo navegar la estructura del proyecto, cómo usar la línea de comandos Artisan y, finalmente, cómo construir tu primer CRUD completo con base de datos real.
Ya tienes experiencia con Node.js, React y Django, lo que te da una ventaja enorme: conoces el patrón MVC, entiendes el concepto de ORM y sabes lo que es un sistema de rutas. Laravel usa exactamente esos mismos conceptos, pero con la sintaxis de PHP y su propio conjunto de herramientas. Verás que muchas cosas te resultarán familiares.
Esta guía está dividida en dos clases de 6 horas cada una. La primera clase cubre la instalación, configuración y fundamentos del framework. La segunda clase es completamente práctica: construirás un CRUD de gestión de estudiantes paso a paso, escribiendo cada línea de código con su explicación.
⚠️ IMPORTANTE
Esta guía está diseñada para seguirse en orden. Cada paso depende del anterior. Si algo no funciona, no avances: resuelve el problema primero, porque los errores de instalación suelen arrastrase a lo largo del proyecto.

3.3  Actividad de apropiación — Instalación paso a paso  (4 horas)
1
Verificar e instalar PHP 8.2+ en Windows
Abre PowerShell o Git Bash y escribe:
php -v
Si ves algo como PHP 8.2.x o superior, ¡ya está instalado! Si no aparece o la versión es menor a 8.1, sigue estos pasos:
Ve a https://windows.php.net/download
Descarga la versión más reciente: PHP 8.2 VS16 x64 Thread Safe (.zip)
Descomprime el archivo en C:\php
Copia el archivo C:\php\php.ini-development, pégalo en la misma carpeta y renómbralo php.ini
Abre php.ini con VS Code y busca (Ctrl+F) estas líneas — quita el punto y coma (;) del inicio para habilitarlas:
;extension=pdo_sqlite   →   extension=pdo_sqlite
;extension=sqlite3      →   extension=sqlite3
;extension=openssl      →   extension=openssl
;extension=mbstring     →   extension=mbstring
;extension=fileinfo     →   extension=fileinfo
;extension=curl         →   extension=curl
;extension=zip          →   extension=zip
Agrega C:\php a la variable de entorno PATH (Panel de control → Sistema → Variables de entorno → Path → Editar → Nuevo → C:\php)
Cierra y vuelve a abrir PowerShell. Ejecuta php -v para verificar.
✅ CONSEJO
Si en la sala ya tienen XAMPP instalado, PHP ya está disponible. Prueba: C:\xampp\php\php.exe -v. En ese caso agrega C:\xampp\php al PATH.
2
Instalar Composer
Composer es el gestor de paquetes de PHP. Sin él no puedes crear proyectos Laravel.
Ve a https://getcomposer.org/download
Descarga Composer-Setup.exe y ejecútalo
El instalador detectará PHP automáticamente (asegúrate de que apunte a C:\php\php.exe)
Finaliza la instalación y verifica en PowerShell:
composer --version
Deberías ver algo como: Composer version 2.x.x
⚠️ IMPORTANTE
Si Composer dice que PHP no está en el PATH, cierra y vuelve a abrir PowerShell. Si sigue sin funcionar, reinicia Windows y vuelve a intentar.
3
Crear el primer proyecto Laravel
Ahora sí: crea tu primer proyecto Laravel. En PowerShell, navega al escritorio o a tu carpeta de proyectos y ejecuta:
cd C:\Users\TuUsuario\Desktop
composer create-project laravel/laravel mi-primer-laravel
Esto descargará Laravel y todas sus dependencias (puede tardar 2-5 minutos según la conexión). Cuando termine:
cd mi-primer-laravel
php artisan serve
Abre tu navegador y ve a: http://localhost:8000
Deberías ver la pantalla de bienvenida de Laravel con el texto 'Laravel' y la versión. ¡Felicitaciones, tu primer proyecto Laravel está vivo!
⚠️ IMPORTANTE
Si aparece el error 'No application encryption key has been set', ejecuta: php artisan key:generate
4
Abrir el proyecto en VS Code y explorar la estructura
Con el servidor corriendo en una terminal, abre una segunda terminal y ejecuta:
code .
VS Code abrirá el proyecto. Instala la extensión recomendada: busca 'Laravel Blade Snippets' en el panel de extensiones (Ctrl+Shift+X).
Ahora explora cada carpeta. Esta es la estructura que más usarás:
Archivo / Carpeta
¿Para qué sirve?
app/Models/
Aquí van los Modelos (clases PHP que representan tablas de la BD)
app/Http/Controllers/
Aquí van los Controladores (lógica de la aplicación)
database/migrations/
Archivos que definen la estructura de las tablas de la BD
resources/views/
Las Vistas: archivos Blade (.blade.php) con el HTML
routes/web.php
Todas las rutas de tu aplicación web van aquí
.env
Variables de entorno: BD, correo, app name, etc. (¡nunca lo subas a Git!)
public/
La carpeta pública: aquí apunta el servidor web
storage/
Archivos generados: logs, caché, uploads
composer.json
Lista de dependencias PHP del proyecto (como package.json en Node)
artisan
El punto de entrada de la CLI de Laravel
5
Configurar SQLite como base de datos
Laravel usa MySQL por defecto, pero nosotros configuraremos SQLite porque no requiere instalar ningún servidor adicional.
Abre el archivo .env en la raíz del proyecto con VS Code y busca la sección de base de datos. Reemplaza todo el bloque DB_* con esto:
DB_CONNECTION=sqlite
# Elimina o comenta estas líneas si existen:
# DB_HOST=127.0.0.1
# DB_PORT=3306
# DB_DATABASE=laravel
# DB_USERNAME=root
# DB_PASSWORD=
Luego crea el archivo de base de datos (Laravel lo espera aquí por defecto):
# En PowerShell dentro de la carpeta del proyecto:
New-Item -ItemType File database\database.sqlite
Verifica que la conexión funciona ejecutando:
php artisan migrate
Deberías ver una lista de migraciones ejecutadas (users, cache, jobs). Si no hay errores, ¡la BD está lista!
✅ CONSEJO
El archivo database.sqlite está dentro de la carpeta database/ del proyecto. Puedes abrirlo con la extensión SQLite Viewer de VS Code para ver las tablas gráficamente.

3.5  Contextualización — El ciclo de una petición en Laravel  (30 min)
Antes de escribir código, entendamos el flujo completo de una petición en Laravel:
#
Quién actúa
¿Qué hace?
1
Navegador
El usuario escribe una URL o hace clic en un enlace: GET /estudiantes
2
routes/web.php
Laravel revisa el archivo de rutas y busca cuál ruta coincide con /estudiantes
3
Controller
La ruta llama al método index() del EstudianteController
4
Model (Eloquent)
El controlador le pide al Modelo todos los estudiantes: Estudiante::all()
5
Base de datos (SQLite)
Eloquent traduce eso a SQL: SELECT * FROM estudiantes
6
Controller
El controlador recibe los datos y los envía a la Vista: return view('estudiantes.index', compact('estudiantes'))
7
View (Blade)
La vista recibe los datos y genera el HTML con @foreach
8
Navegador
El HTML llega al navegador y el usuario ve la lista de estudiantes
3.6  Actividad de apropiación — Construyendo el CRUD paso a paso  (4.5 horas)
1
Crear el Modelo, Migración y Controlador (con un solo comando)
Ejecuta este comando en la terminal desde la carpeta del proyecto:
php artisan make:model Estudiante -mcr
Este comando crea tres archivos de una vez:
app/Models/Estudiante.php — El Modelo
database/migrations/xxxx_create_estudiantes_table.php — La Migración
app/Http/Controllers/EstudianteController.php — El Controlador con los 7 métodos RESTful
💡 NOTA
El flag -m crea la migración, -c crea el controlador y -r lo hace 'resource' (con todos los métodos CRUD ya definidos vacíos).
2
Definir la estructura de la tabla en la Migración
Abre el archivo de migración generado en database/migrations/. Su nombre termina en _create_estudiantes_table.php. Edita el método up():
public function up(): void
{
    Schema::create('estudiantes', function (Blueprint $table) {
        $table->id();                          // ID autoincremental
        $table->string('nombre');              // Nombre del estudiante
        $table->string('apellido');            // Apellido
        $table->string('email')->unique();     // Email (único, no se repite)
        $table->string('ficha');               // Número de ficha SENA
        $table->string('programa');            // Programa de formación
        $table->timestamps();                  // created_at y updated_at
    });
}
Luego ejecuta la migración para crear la tabla en SQLite:
php artisan migrate
✅ CONSEJO
Si necesitas hacer cambios a la migración después de ejecutarla, usa php artisan migrate:rollback para deshacerla, edita el archivo y vuelve a ejecutar php artisan migrate.
3
Configurar el Modelo Estudiante
Abre app/Models/Estudiante.php y agrega la propiedad $fillable. Esto le dice a Laravel qué campos se pueden llenar masivamente (protección contra asignación masiva):
<?php
namespace App\Models;
use Illuminate\Database\Eloquent\Factories\HasFactory;
use Illuminate\Database\Eloquent\Model;
class Estudiante extends Model
{
    use HasFactory;
    protected \$fillable = [
        'nombre',
        'apellido',
        'email',
        'ficha',
        'programa',
    ];
}
⚠️ IMPORTANTE
Si no defines $fillable, Laravel lanzará un error MassAssignmentException cuando intentes guardar datos. Siempre declara los campos que aceptas.
4
Registrar las rutas RESTful en web.php
Abre routes/web.php y agrega esta línea. Una sola línea crea las 7 rutas del CRUD automáticamente:
<?php

use Illuminate\Support\Facades\Route;
Route::get('/', function () {
    return view('welcome');
});
// Esta línea registra las 7 rutas RESTful del CRUD:
Route::resource('estudiantes', EstudianteController::class);
Verifica las rutas generadas ejecutando:

5
Programar el Controlador — los 4 métodos del CRUD
Abre app/Http/Controllers/EstudianteController.php. Reemplaza el contenido completo con este código:
<?php
namespace App\Http\Controllers;
use App\Models\Estudiante;
use Illuminate\Http\Request;
class EstudianteController extends Controller
{
    // READ — Listar todos los estudiantes
    public function index()
    {
        \$estudiantes = Estudiante::all();
        return view('estudiantes.index', compact('estudiantes'));
    }
    // CREATE — Mostrar formulario de creación
    public function create()
    {
        return view('estudiantes.create');
    }
    // CREATE — Guardar el nuevo estudiante en la BD
    public function store(Request \$request)
    {
        \$request->validate([
            'nombre'   => 'required|min:2|max:100',
            'apellido' => 'required|min:2|max:100',
            'email'    => 'required|email|unique:estudiantes',
            'ficha'    => 'required',
            'programa' => 'required',
        ]);
        Estudiante::create(\$request->all());
        return redirect()->route('estudiantes.index')
                         ->with('success', 'Estudiante registrado correctamente.');
    }
    // UPDATE — Mostrar formulario de edición
    public function edit(Estudiante \$estudiante)
    {
        return view('estudiantes.edit', compact('estudiante'));
    }
    // UPDATE — Guardar los cambios en la BD
    public function update(Request \$request, Estudiante \$estudiante)
    {
        \$request->validate([
            'nombre'   => 'required|min:2|max:100',
            'apellido' => 'required|min:2|max:100',
            'email'    => 'required|email|unique:estudiantes,email,'
                          .\$estudiante->id,
            'ficha'    => 'required',
            'programa' => 'required',
        ]);
        \$estudiante->update(\$request->all());
        return redirect()->route('estudiantes.index')
                         ->with('success', 'Estudiante actualizado correctamente.');
    }
    // DELETE — Eliminar el estudiante
    public function destroy(Estudiante \$estudiante)
    {
        \$estudiante->delete();
        return redirect()->route('estudiantes.index')
                         ->with('success', 'Estudiante eliminado correctamente.');
    }
}
💡 NOTA
Route Model Binding: cuando el método recibe Estudiante $estudiante como parámetro, Laravel automáticamente busca el estudiante por ID en la BD. No necesitas escribir Estudiante::find($id) manualmente.

5.  GLOSARIO DE TÉRMINOS
Término
Definición
Laravel
Framework PHP de código abierto basado en el patrón MVC, creado por Taylor Otwell en 2011. Ofrece herramientas integradas para routing, ORM, autenticación, caché y más.
Composer
Gestor de dependencias para PHP (equivalente a npm en Node.js). Permite instalar, actualizar y gestionar los paquetes que usa un proyecto PHP.
Artisan
Interfaz de línea de comandos (CLI) de Laravel. Automatiza tareas como crear archivos, ejecutar migraciones, limpiar caché y arrancar el servidor.
MVC
Patrón de arquitectura Model-View-Controller. Separa los datos (Model), la presentación (View) y la lógica (Controller) para organizar mejor el código.
Eloquent ORM
ORM de Laravel que permite interactuar con la base de datos usando objetos PHP en lugar de SQL. Cada tabla tiene un Modelo correspondiente.
Blade
Motor de plantillas de Laravel. Permite escribir vistas PHP con directivas limpias: @extends, @section, @foreach, @if, {{ $variable }}.
Migración
Archivo PHP que define los cambios en la estructura de la BD (crear, modificar o eliminar tablas). Funciona como control de versiones para la base de datos.
Route Model Binding
Función de Laravel que inyecta automáticamente el objeto del modelo como parámetro del controlador a partir del ID de la URL, sin necesidad de hacer la consulta manualmente.
$fillable
Propiedad del Modelo que lista los campos que se pueden asignar masivamente. Es una medida de seguridad contra la asignación masiva no autorizada.
SQLite
Motor de base de datos embebido que almacena todos los datos en un único archivo .sqlite. No requiere servidor separado, ideal para desarrollo y aprendizaje.
@csrf
Directiva Blade que inserta un token de seguridad en los formularios para proteger contra ataques CSRF (Cross-Site Request Forgery).
@method('PUT')
Directiva Blade que simula métodos HTTP (PUT, PATCH, DELETE) en formularios, ya que HTML nativo solo soporta GET y POST.
